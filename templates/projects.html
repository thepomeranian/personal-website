One of the many learning opportunities, other than building the entire application from the ground up, was how information flows from the backend to the frontend. When I was trying to use Isotope/Masonry to filter and sort tv shows by genre, I didn’t want to have multiple query statements to get all of the genre information for every tv show and utilized the many to many relationship to get all of the information using one query statement. As an added bonus, that dramatically reduced page load time since the query only needed to happen once. One of the things I really struggled with was with my database model. It was important to me to make everything easy to access and learning now just how to use many-many relationships, but also how it works and why it works. I was able to utilize the many to many relationship to query information about a single tv show from the an episode summary from a particular season to the user that has it(tv show) favorited.
